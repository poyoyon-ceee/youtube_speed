# 調査メモ：登録チャンネル → 動画タブ（本チャットの試行錯誤まとめ）

本ファイルは、**チャット内で議論・実装・切り分けした内容**をルートに回収したもの。  
「回避予定の不具合／未解決事象」と、**原因仮説の整理**、**試したことの境界**を書いておく。

---

## 1. 当初の課題（ユーザー報告）

- **チャンネル「動画」タブ自動遷移**（`handleRedirect`）利用時、チャンネルに入ると**一瞬ホーム相当の画面／URL が見えてから**動画タブに切り替わる。**ワンテンポ遅い**。
- 普段の導線は**外部から特定動画へ飛ぶことは少なく**、主に**左の「登録チャンネル」**から入る。

---

## 2. 原因の整理（議論で出た論点）

| 論点 | 内容 |
|------|------|
| コンテンツスクリプトの注入タイミング | 既定は `document_idle` に近く、**表示が始まったあと**に処理が走りやすい。 |
| SPA | YouTube は **内部ルータ**で遷移するため、「URL を後から `replace`」は**一瞬ホーム URL**がアドレスバーに出るのと整合しやすい。 |
| 対策案（議論のみ含む） | **早い `run_at`**、**declarativeNetRequest**、**クリック先取り**、**ガイド内 `href` 書き換え** など。 |

---

## 3. 実装したこと（本チャットの Agent 実装）

**目的**：左ガイドの登録チャンネル相当リンクを、**クリック前に** `…/videos` 付き URL にして、ホームの一瞬を減らす。

**対象コード**：`src/content.js`（バージョンは `manifest.json` / `PROJECT.md` で **0.2.1** 付近に更新済み）

**主な追加**：

- `channelHomeToVideosUrl(fullHref)` … チャンネルホーム相当パスを `…/videos` に変換（`/@…`、`/channel/UC…`、`/c/…`、`/user/…`、`/featured` 等）。
- `getGuideLinkRoots()` … `#guide-inner-content` と `ytd-mini-guide-renderer` を走査ルートに。
- `rewriteSubscriptionGuideLinks()` / `scheduleRewriteSubscriptionGuideLinks()`（デバウンス約 120ms）  
  - `autoVideoTab` ON のとき：`href` を書き換え、元を `data-yt-speed-orig-href` に保持。  
  - OFF のとき：復元。
- 呼び出し：`init`、`MutationObserver`、`yt-navigate-finish`、`chrome.storage.onChanged` 経路など。

**ポップアップ**：トグルは従来どおり **「チャンネル時に『動画』タブへ」**（`autoVideoTab`）と連動。項目は増やしていない。

**ドキュメント追記**：`PROJECT.md`、`docs/walkthrough_auto_redirect.md`。

**従来機能**：`handleRedirect()`（`/@…` 等でホームに来たあと `location.replace` で `/videos`）は **リポジトリ上は残っている**。

---

## 4. 観測された現象（ユーザー試験）

### 4.1 右クリック「リンクの URL をコピー」

- コピー結果に **`/videos` が付いていない**ことがある。

**想定原因（チャット上の整理）**：

1. YouTube が **再描画で `href` を元のホーム用に戻す**（拡張の書き換えと内部データのどちらが勝つかの競合）。
2. 右クリック対象が **`<a>` 以外**で、コピー元が別の URL になっている。

### 4.2 アドレスバー：ホーム URL → 少しして `/videos`

- 登録チャンネルをクリックすると、アドレスバーが**いったんホーム相当**になり、**ワンクッションあと** `/videos` が付くように見える。

### 4.3 切り分け：`handleRedirect` を外して試した結果

- **ユーザー環境で** `handleRedirect` を外して試したが、**挙動はほぼ変わらなかった**（チャンネル「ホーム」側の表示／URL の流れは残る）。

**解釈（チャット上の結論）**：

- 上記二段表示の主因は、**少なくとも `handleRedirect` だけでは説明しきれない**。
- **YouTube の SPA が、ガイドのクリックに対して DOM の `href` だけに従わず、内部の正規 URL（ホーム相当）で先に遷移している**可能性が高い。  
  → **`href` 書き換えだけでは「最初から動画タブ」に寄せきれない**ケースがある、という切り分け。

※リポジトリの現状ブランチでは `handleRedirect` は**まだ有効**なコードとして残っている。ローカル試験での削除は**マージされていない試行**として扱う。

## 5. 未回収・回避予定の事象（いまの実装の限界）

| 事象 | 状態 |
|------|------|
| 登録チャンネルクリック後も **ホーム → 動画** の段階的表示／体感 | **未解決**（`href` 書き換え＋現行 `handleRedirect` ではユーザー環境で十分に解消できていない）。 |
| 右クリックコピーで **`/videos` が付かない** | **未解決**（上記と同根の可能性：DOM 上書き・コピー元の要素差）。 |
| ブックマークバー直クリック等 **YouTube 外からの遷移** | 本拡張のガイド書き換えの対象外（当初からスコープ外の議論あり）。 |

---

## 6. 今後の回避案（未実装・チャットで挙がった方向）

優先度は要相談。いずれも**追加の実装・検証**が必要。

1. **`click` / `pointerdown` のキャプチャ**で、対象がチャンネルホーム相当なら **`preventDefault` して `…/videos` へ自前遷移**（YouTube の既定ナビを横取り）。副作用・壊れやすさの検証が必要。
2. **`declarativeNetRequest` 等**で、特定 URL パターンを **`/videos` にリダイレクト**（権限・ルール・例外 URL の設計が必要）。
3. **`run_at: "document_start"`** などで、フルロード時の後段処理を少し早める（根本解決にはなりにくいが、補助になり得る）。
